# Open Assistant

[English](README.md)

AI 驱动的代码与文档补全助手。截取屏幕画面，发送给 AI 助手（如豆包桌面端），并等待 AI 自动将回复插入到光标位置。

## 工作原理

1. 配置一个或多个 AI Agent，编写提示词
2. 为每个想用的提示词设置一个全局快捷键
3. 随时按下快捷键——助手截图、发送给指定 Agent 和提示词，并等待 AI 将回复插入到光标位置

## 功能特点

- **多 Agent 支持** — 在同一界面管理多个 AI Agent（豆包桌面端等）
- **按提示词设快捷键** — 每个提示词拥有独立的全局快捷键，无需先选择再触发
- **两种捕获模式**
  - **SSE Fetch** — 在网络层拦截 AI 回复，窗口最小化到托盘时也能正常工作
  - **DOM Poll** — 在豆包聊天页面中通过 DOM 轮询获取回复
- **两种输出模式**
  - **Streaming（流式）** — 随 AI 生成逐段粘贴
  - **Full（完整）** — 等待完整回复后一次粘贴
- **流式剪贴板** — 粘贴完成后恢复原始剪贴板内容
- **系统托盘** — 静默运行于系统托盘，无需保持窗口打开
- **自动检测安装路径** — 启动时自动搜索豆包可执行文件位置
- **自定义提示词** — 在设置界面中创建、编辑、删除提示词模板

## 快速开始

```bash
# 安装依赖
npm install

# 开发模式启动（含 DevTools）
npm run dev

# 正常启动
npm start
```

### 构建分发版本

```bash
npm run build

npm run build:win

npm run build:linux
```

## 使用方法

1. 从系统托盘打开设置窗口
2. 在 Agent 卡片中设置 CDP Endpoint（例如 `http://127.0.0.1:9225`），不同 Agent 可使用不同端口号，示例中的端口仅为示意
3. 在 Agent 卡片上点击 **初始化**，或在托盘菜单中点击 **初始化豆包**，以正确调试端口启动 Agent
4. 在每个 Agent 卡片中，点击提示词胶囊上的 ✏️ 图标设置全局快捷键
5. 在任何应用中按下快捷键即可触发助手

## 常见问题

### 无法连接到豆包桌面端
- 请使用 Agent 卡片中的**初始化**按钮，或托盘菜单中的**初始化豆包**，以正确调试端口启动 Agent
- 检查端口是否被防火墙拦截

### 快捷键无效
- 检查是否与其他软件的快捷键冲突
- 尝试使用不同的快捷键组合
- macOS 用户需要为应用授予辅助功能权限

### 无法自动粘贴
- 确保设置中的 **Auto Insert** 已启用
- 确保等待回复的过程中您正在使用的编辑器没有失去焦点
